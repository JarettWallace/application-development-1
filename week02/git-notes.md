* Local repo is stored on one individual machine
    * A remote repo is stored on some type of server


* Repo or repository is a form of project storage


* Commits are updates/notes made after changes to document what changes have been made, they should be easy to understand and include all changes


* Branching is changing from your main line of development to a different line in order to preserve your original work while being able to work on something else without messing with that main line


* A staging area is the waiting room for changes where you are able to tell GIT what to include in the next commit


* Git push, a command that is used to upload your local repository into a remote repository, it is also required for submitting assignments in this class


* Git pull is the opposite of push as it is used to download content from a remote repo and update your local one to match what was downloaded


* gitignore is used to tell git what types of files it can and should ignore
    * is very useful in making sure accidental commits do not occur


* README is a document that helps introduce the project or assignment
    * can include things such as a description, installation steps, examples of usage, any relevant licensing, and possibly contact information of the owner of the repo
